# public/branding-samples/

Изображения вариантов нанесения (вышивка, шелкография, DTF и т.д.).

## Структура

```
public/branding-samples/
├── vyshivka/           # Вышивка (6 файлов)
├── shevrony/           # Шевроны (5 файлов)
├── shelkografiya/      # Шелкография (7 файлов)
├── dtf-pechat/         # DTF-печать (5 файлов)
├── dtg-pechat/         # DTG-печать (4 файла)
├── sublimaciya/        # Сублимация (4 файла)
├── tisnenie/           # Тиснение (4 файла)
├── gravirovka/         # Гравировка (5 файлов)
├── birki/              # Бирки и стикеры (6 файлов)
└── sources.json        # Источники и лицензии всех картинок
```

**Полный список нужных файлов с описаниями и поисковыми запросами:** [docs/BRANDING_IMAGE_TODO.md](../../docs/BRANDING_IMAGE_TODO.md)
**Таблица источников:** [docs/BRANDING_IMAGE_SOURCES.md](../../docs/BRANDING_IMAGE_SOURCES.md)

## Требования к файлам
- Формат: `.webp` (приоритет), допускается `.jpg`/`.png`.
- Ширина: 1200–1600 px.
- Вес: до 600 KB.
- Имя файла: латиницей, через дефис, без пробелов.
- Без водяных знаков и видимых чужих логотипов / лиц крупным планом.

## Как добавить картинку
- **Через админку (рекомендуется):** `/admin/catalog-editor/` → вкладка «Нанесение» → выбрать метод → редактировать вариант → загрузить файл. Файл попадёт в `/uploads/branding/` и подставится автоматически.
- **Через файловую систему:** положить файл с нужным именем в нужную подпапку — путь уже прописан в `lib/brandingSamples.ts`, картинка появится после обновления страницы.

После добавления — **обязательно** дополнить `sources.json` и `docs/BRANDING_IMAGE_SOURCES.md`.
