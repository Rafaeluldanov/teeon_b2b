# 11. SEO и Яндекс

## Что реализовано

### Базовые мета-теги
- `<title>` — уникальный для каждой страницы, шаблон: `Заголовок страницы | TEEON`
- `<meta name="description">` — уникальное описание 120–160 символов
- `<link rel="canonical">` — канонический URL страницы
- `<meta name="robots">` — управление индексацией (`index/noindex`)

### Open Graph
- `og:title`, `og:description` — для превью в соцсетях и мессенджерах
- `og:image` — OG-изображение (`public/og-default.svg`, нужно заменить на реальное 1200×630 px)
- `og:type`, `og:url`, `og:locale` (ru_RU), `og:site_name`
- `twitter:card: summary_large_image`

### Sitemap
Динамический `sitemap.xml` — файл `app/sitemap.ts`. Включает все публичные страницы с `lastModified`, `changeFrequency` и `priority`. Обновляется автоматически при добавлении новых кейсов или категорий.

URL: `https://teeon.ru/sitemap.xml`

### Robots.txt
Файл `app/robots.ts`. Разрешает всё, закрывает `/admin/` и `/api/admin/`. Указывает ссылку на sitemap.

URL: `https://teeon.ru/robots.txt`

## JSON-LD схемы

JSON-LD разметка подключена на всех страницах через компонент `JsonLd`.

### Схемы на корневом уровне (все страницы)
Добавлены в `app/layout.tsx`:
- **Organization** — название, URL, контактный email и телефон
- **WebSite** — название сайта, URL, потенциал поиска (SearchAction)

### Схемы на отдельных страницах
| Страница | Схема |
|---|---|
| Главная `/` | FAQPage |
| Страницы категорий каталога | Service (услуга пошива) |
| Страницы брендирования | Service (услуга брендирования) |
| Страницы портфолио | BreadcrumbList |
| Страница о компании | BreadcrumbList |

### Почему не используется Product/Offer без цен

Схема `Product` с `Offer` требует указания цены. Так как цены не фиксированы (зависят от тиража, ткани, нанесения), использование `Product` без цены создаёт невалидную разметку. Поэтому используется `Service` — это соответствует реальной бизнес-модели.

## SEO-конфигурация

### Где менять базовые SEO-настройки

Файл: `lib/seo.ts`
```ts
export const siteConfig = {
  name: 'TEEON',
  descriptor: 'Пошив промо-одежды и корпоративного мерча под ключ',
  url: process.env.NEXT_PUBLIC_SITE_URL ?? 'https://teeon.ru',
  defaultTitle: 'Пошив промо-одежды и мерча на заказ | TEEON',
  defaultDescription: '...',
  ogImage: '/og-default.svg',
}
```

### Где задавать домен

`.env.local`:
```env
NEXT_PUBLIC_SITE_URL=https://teeon.ru
```

### Где задавать Яндекс.Метрику

`.env.local`:
```env
NEXT_PUBLIC_YANDEX_METRIKA_ID=12345678
```

**Что проверяется:**
- Наличие `<title>` (10–70 символов)
- Наличие `<meta name="description">` (80–160 символов)

- Наличие `og:title` и `og:description`

- HTTP-статус 200

## Что проверить после деплоя

### В Яндекс.Вебмастере (webmaster.yandex.ru)

1. Добавить сайт → подтвердить права владения (HTML-файл или meta-тег)
2. Добавить sitemap: `https://teeon.ru/sitemap.xml`
3. Проверить раздел «Индексирование» — нет ли заблокированных страниц
4. Проверить раздел «Ошибки» — нет ли 404, 500
5. Настроить уведомления о технических ошибках

### В Яндекс.Метрике (metrika.yandex.ru)

1. Создать счётчик → получить ID
2. Добавить ID в `.env.local`: `NEXT_PUBLIC_YANDEX_METRIKA_ID=12345678`
3. Пересобрать и задеплоить сайт
4. Проверить что счётчик срабатывает: открыть сайт → в Метрике должно появиться посещение
5. Настроить цели: нажатие кнопки «Получить расчёт», успешная отправка формы
