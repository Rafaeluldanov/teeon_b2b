| Команда | Что проверяет | Нужен сервер |

## npm run lint

Запускает ESLint с конфигурацией из `.eslintrc.json`. Проверяет:
- Синтаксические ошибки TypeScript
- Правила Next.js (неиспользуемые импорты, `<img>` вместо `<Image>`, и др.)
- Запрещённые паттерны

**Статус:** ✅ 0 ошибок (последняя проверка 18.05.2026)

---

## npm run build

Production-сборка Next.js. Проверяет:
- TypeScript-типы по всему проекту
- Корректность импортов и экспортов
- Статическую генерацию всех страниц
- Метаданные (metadata) на каждой странице

Результат сборки: 44+ страниц, 0 ошибок.

**Что делать при ошибке сборки:**
1. Прочитать вывод — ошибка указывает файл и строку
2. Исправить проблему
3. Повторить `npm run build`

## npm run test:form

Тестирует API-эндпоинт `/api/request` через прямые HTTP-запросы (без браузера).

Три сценария:
1. **Базовая отправка** — ФИО, телефон, тема → должен вернуть `success: true`
2. **Отправка с файлом** — те же данные + тестовый файл `tests/fixtures/test-logo.txt`
3. **Ethereal Email** — если `LEAD_EMAIL_MODE=ethereal`, проверяет наличие `previewUrl` в ответе

**Статус:** ✅ 3/3 (последняя проверка 18.05.2026)

## npm run seo:audit

- JSON-LD — наличие и валидность JSON

## Что делать при типичных ошибках

### npm run build падает с TypeError

Проверьте импорты — возможно, импортируется несуществующий файл или функция.

### npm run seo:audit показывает 500

Проверьте, запущен ли сервер (`npm run dev`). Если запущен — проблема на конкретной странице. Смотрите вывод ошибки в консоли сервера.

### E2E-тесты падают с «page.goto timeout»

Сервер не запущен или занят. Проверьте `npm run dev`.

### Stale .next cache

```bash
rm -rf .next
npm run dev
```
