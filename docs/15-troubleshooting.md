# 15. Типичные проблемы и решения

## Проблемы с установкой

### brew: command not found

Homebrew не установлен. На macOS можно установить Node.js напрямую с nodejs.org без Homebrew.

```
Скачать: https://nodejs.org/en → кнопка «LTS»
```

### node/npm не найдены после установки

Перезапустите терминал после установки Node.js. Если не помогает — установка прошла некорректно, переустановите Node.js.

```bash
node -v   # должно показать версию
npm -v    # должно показать версию
```

## Проблемы с запуском сервера

### missing required error components / Application error: a client-side exception has occurred

Устаревший кэш сборки. Решение:
```bash
rm -rf .next
npm run dev
```

### Cannot find module './948.js' или похожие ошибки

Повреждён кэш или `node_modules`. Решение:
```bash
rm -rf .next node_modules
npm install
npm run dev
```

### Порт 3000 уже используется

```bash
lsof -ti:3000 | xargs kill -9
npm run dev
```

### EACCES: permission denied на .next/

```bash
sudo chown -R $(whoami) .next
npm run dev
```

## Проблемы с переменными окружения

### admin variables not configured

В `.env.local` не заданы `ADMIN_USERNAME`, `ADMIN_PASSWORD` или `ADMIN_SESSION_SECRET`. Убедитесь, что:

1. Файл `.env.local` существует в корне проекта (рядом с `package.json`)
2. Все три переменные заполнены
3. Сервер был перезапущен после изменения `.env.local`

### Сайт показывает неправильный домен в sitemap

Проверьте `NEXT_PUBLIC_SITE_URL` в `.env.local`. Значение должно быть без слэша в конце:
```
NEXT_PUBLIC_SITE_URL=https://teeon.ru   ← правильно
NEXT_PUBLIC_SITE_URL=https://teeon.ru/  ← неправильно
```

### Яндекс.Метрика не работает

1. Проверьте, что `NEXT_PUBLIC_YANDEX_METRIKA_ID` заполнен числом (не пустой)
2. Пересоберите проект: `npm run build`
3. Откройте сайт в браузере без блокировщиков рекламы

## Проблемы с авторизацией в админке

### 401 Unauthorized при загрузке изображений в AdminCatalogEditor

**Причина:** в браузере осталась старая cookie сессии с `path: '/admin'`. Такая cookie не отправляется на `/api/admin/upload`, поэтому возникает ошибка авторизации.

**Решение:**

1. Нажмите кнопку **«Выйти»** в верхней панели AdminCatalogEditor.
2. Войдите заново — введите логин и пароль на `/admin/login/`.
3. После входа cookie будет создана с корректным `path: '/'` и отправляться на все `/api/admin/*` маршруты.

**Если Выйти не помогло (ghost-cookie):**

1. Откройте DevTools браузера (F12 → вкладка Application → Cookies → localhost:3000).
2. Найдите cookie с именем `teeon_admin_session` и удалите все записи с этим именем.
3. Перезапустите сервер: `npm run dev`.
4. Войдите в adminку заново.

---

### Неверный логин или пароль

Проверьте `ADMIN_USERNAME` и `ADMIN_PASSWORD` в `.env.local`. Значения чувствительны к регистру и пробелам.

### После входа редирект обратно на login

Проблема с cookie. Возможные причины:
- `ADMIN_SESSION_SECRET` слишком короткий или содержит недопустимые символы
- Сервер не перезапустился после изменения `.env.local`

Сгенерируйте новый секрет:
```bash
openssl rand -base64 32
```

### Загруженное фото варианта не отображается на публичной странице каталога

1. Убедитесь, что загрузили фото именно в форму **варианта** (не модели).  
   Путь: категория → модель → «Редактировать вариант» → поле «Изображение варианта».
2. Нажмите **«Сохранить вариант»** → затем **«Сохранить изменения»** (форма модели) → только тогда данные попадут в localStorage.
3. Обновите публичную страницу (например `/catalog/futbolki/`) — компонент читает localStorage при загрузке.
4. Если фото не появилось: DevTools → Application → Local Storage → `teeon_admin_catalog_models` → найдите нужный вариант → проверьте поле `image`.

> **Изображение добавляется только к варианту.** У категории и модели поля изображения в UI нет.

### Фото портфолио не отображается на сайте

1. Откройте вкладку «Портфолио» → выберите кейс → загрузите **coverImage** и/или **galleryImages**.
2. Нажмите **«Сохранить кейс»**.
3. Обновите `/portfolio/` или `/portfolio/[slug]/` — страницы читают `teeon_admin_portfolio_cases` из localStorage.
4. Если не помогло: DevTools → Application → Local Storage → `teeon_admin_portfolio_cases` → найдите кейс по slug → проверьте поля `coverImage`, `galleryImages`.

## Проблемы с формой

### Форма не отправляется (кнопка не реагирует)

Возможная причина: React ещё не завершил гидрацию. Дождитесь появления атрибута `data-ready="true"` на форме (это происходит за долю секунды после загрузки страницы).

### Файл не прикрепляется

Проверьте:
- Формат файла: PDF, PNG, JPG, WEBP, SVG, TXT, DOC, DOCX, XLS, XLSX, ZIP
- Размер: не более 10 МБ

### Заявка не приходит на email

1. Убедитесь, что `LEAD_EMAIL_MODE=smtp` (не `dev`)
2. Проверьте SMTP-настройки в `.env.local`
3. Проверьте папку «Спам» в почтовом клиенте
4. Запустите `npm run test:form` для диагностики

### В режиме dev заявки не сохраняются

Ищите JSON-файлы в папке `.tmp/request-submissions/`. Папка создаётся при первой отправке. Если папки нет — значит отправки не было.

## Проблемы со сборкой

### npm run build падает с TypeScript-ошибкой

Прочитайте вывод ошибки — Next.js указывает файл и строку. Типичные причины:
- Неправильный тип данных
- Несуществующее поле в объекте
- Неверный импорт

### npm run build падает с «Module not found»

Проверьте импорты в указанном файле. Возможно, файл был переименован или удалён.

### npm run lint показывает ошибки

Исправьте ошибки согласно описанию ESLint. Типичные проблемы:
- `no-unused-vars` — удалите неиспользуемую переменную
- `@next/next/no-img-element` — используйте `<Image>` вместо `<img>`

## Проблемы с MinIO (хранилище изображений)

### Загрузка изображений в админке не работает

Убедитесь, что MinIO запущен:
```bash
# Проверить доступность
curl http://127.0.0.1:9000/minio/health/live
```

Если не отвечает — запустите MinIO:
```bash
MINIO_ROOT_USER=minioadmin MINIO_ROOT_PASSWORD=minioadmin \
./minio server ~/minio-data --console-address ":9001"
```

### Бакет не существует

Откройте консоль MinIO (http://localhost:9001) → создайте бакет `teeon-images` → установите Access Policy → Public.

Или используйте mc (MinIO Client):
```bash
/tmp/mc alias set local http://127.0.0.1:9000 minioadmin minioadmin
/tmp/mc mb local/teeon-images
/tmp/mc anonymous set public local/teeon-images
```

## Проблемы с контентом

### Изменения в lib/*.ts не применяются

Перезапустите сервер разработки: `Ctrl+C` → `npm run dev`. Hot-reload иногда не подхватывает изменения в файлах данных.

### Новая категория не появляется в sitemap

Убедитесь, что новая категория добавлена в `lib/catalog.ts` и экспортируется в массиве `catalogCategories`. Sitemap генерируется автоматически из этого массива.

### Изменения в adminCatalogEditor видны только мне

Это ожидаемое поведение — редактор сохраняет в `localStorage` только текущего браузера. Экспортируйте JSON и перенесите в соответствующий файл в `lib/`.
