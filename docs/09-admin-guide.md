# 09. Руководство по административной части

## Доступ

| Что | Адрес |
|---|---|
| Страница входа | `/admin/login/` |
| Редактор каталога | `/admin/catalog-editor/` |

Логин/пароль задаются в `.env.local` через `ADMIN_USERNAME` / `ADMIN_PASSWORD`.

---

## Как войти / выйти

1. Перейдите на `/admin/login/`, введите логин и пароль.
2. Для выхода — кнопка **«Выйти»** в верхней панели.

---

## Как устроена авторизация

Используется httpOnly-cookie с HMAC-SHA256 подписью (срок — 8 часов). Middleware (`middleware.ts`) защищает все маршруты `/admin/*`. API-маршруты `/api/admin/*` проверяют cookie самостоятельно.

---

## Интерфейс редактора каталога

Редактор построен в **три колонки**:

```
| Категории | Модели категории | Форма редактирования |
```

На мобильных — одна колонка сверху вниз.

---

## Шаг 1. Выбрать категорию

В левой колонке отображаются все категории. Нажмите на категорию — в центре появится список её моделей.

Кнопка **⚙** рядом с категорией — открывает форму редактирования категории.

### Как добавить новую категорию

1. Нажмите **«+ Добавить»** в верхней части колонки категорий.
2. Заполните: Название, Slug (латиницей), Описание, Активность, Сортировка.
3. Нажмите **«Создать категорию»**.

> **Важно:** новая категория появится в редакторе, но публичной страницы `/catalog/<slug>/` не создаётся автоматически. Для этого нужен route в `app/catalog/<slug>/page.tsx` или CMS/backend. Текущие 8 категорий (`futbolki`, `hudi` и др.) продолжают работать без изменений.

---

## Шаг 2. Работа с моделями

В центральной колонке — список моделей выбранной категории. Каждая карточка показывает:
- Миниатюру (если есть изображение)
- Название модели
- Код лекала (если задан)
- Количество вариантов

Нажмите на карточку — справа откроется форма редактирования.

### Как создать новую модель

1. Выберите категорию.
2. Нажмите **«+ Модель»**.
3. Заполните форму (см. поля ниже).
4. Нажмите **«Создать модель»**.

### Поля модели

| Поле | Описание |
|---|---|
| Название | Отображается на публичных страницах |
| Slug | Уникальный идентификатор латиницей |
| Код лекала / артикул | Например `F-BASIC-001` — видно закупщику и на публичной карточке |
| Бейджи | Например: Хит, Новинка (через запятую) |
| Краткое описание | Одна строка, для списка моделей |
| Полное описание | Подробный текст |
| Особенности, Подходит для, Брендирование, Что можно настроить | Списки, по одному пункту на строку |
| Активна | Если выключена — не показывается на сайте |
| Сортировка | Порядок в списке |

> **Изображения модели:** изображение добавляется **только к конкретному варианту** (не к модели в целом). Для добавления фото откройте форму варианта → поле «Изображение варианта».

### Кнопки в форме модели

- **«Создать модель»** / **«Сохранить изменения»** — явное сохранение
- **«Дублировать»** — создать копию с новым ID
- **«Удалить модель»** — удалить безвозвратно
- **«Отмена»** — закрыть форму без сохранения

После сохранения появляется уведомление: «Модель сохранена».

---

## Шаг 3. Варианты внутри модели

В нижней части формы модели — секция **«Варианты модели»**.

Каждый вариант — это отдельный товар (например, разная плотность ткани).

### Как добавить вариант

1. В секции «Варианты модели» нажмите **«+ Добавить вариант»**.
2. Откроется форма варианта (заменяет форму модели).
3. Заполните поля и нажмите **«Сохранить вариант»**.

### Поля варианта

| Поле | Описание |
|---|---|
| Название варианта | Например: `Standard 160 г/м²` |
| Подзаголовок | Короткая пометка |
| Код лекала / артикул | Например `F-OVERSIZE-220` |
| **Изображение варианта** | **URL или загрузка файла — основное место для фото** |
| Описание, Характеристики, Подходит для | Текстовые поля |
| Материалы, Плотности, Размеры | Списки (по строке) |
| Брендирование, Места нанесения, Что можно настроить | Списки (по строке) |
| Цвета | Визуальный редактор: color picker + hex + название |
| Активен, Сортировка | Управление видимостью и порядком |

### Кнопки в форме варианта

- **«Сохранить вариант»** — сохранить и вернуться к форме модели
- **«Дублировать»** — создать копию варианта
- **«Удалить»** — удалить вариант
- **«Отмена»** — вернуться к модели без изменений

---

## Как добавить изображение к варианту

Изображение добавляется **только к варианту** (не к категории и не к модели в целом). Это позволяет показывать разные фото при переключении вариантов на публичной странице.

1. Откройте нужную категорию → модель → нажмите **«Редактировать»** на варианте.
2. Найдите поле **«Изображение варианта»**.
3. Добавьте URL вручную или нажмите кнопку **📁 Загрузить** для загрузки файла.
4. Preview появится сразу под полем.
5. Нажмите **«Сохранить вариант»**.

### Кнопки в блоке изображения

| Кнопка | Действие |
|---|---|
| 📁 Загрузить | Загрузить файл (JPEG, PNG, WEBP, SVG до 10 МБ) |
| Удалить изображение | Очистить поле, убрать preview |
| Открыть | Открыть текущий URL в новой вкладке |

### Ошибки загрузки

| Сообщение | Причина |
|---|---|
| «Сессия истекла, войдите заново» | Истёк токен авторизации — войдите снова |
| «Недопустимый тип файла» | Формат не поддерживается |
| «Файл слишком большой» | Превышен лимит 10 МБ |

### Куда попадает файл

**Если MinIO/S3 настроен** — файл загружается в облако, возвращается публичный URL.

**Если MinIO/S3 не настроен** (локальная разработка) — файл сохраняется в `public/uploads/catalog/` и доступен по URL `/uploads/catalog/<filename>`.

### Как удалить изображение

1. В форме варианта найдите поле «Изображение варианта».
2. Нажмите кнопку **«Удалить изображение»** под preview.
3. Изображение исчезнет из preview — поле очистится.
4. Нажмите **«Сохранить вариант»**.
5. На публичной странице отобразится CSS-заглушка вместо фото.

---

## Как изменения попадают на публичный сайт

Все изменения сохраняются в **localStorage браузера** (`teeon_admin_catalog_models`).

Публичные страницы каталога (`/catalog/futbolki/` и др.) при загрузке читают этот ключ из localStorage и применяют его поверх данных из кода. Таким образом:
- Изменение в редакторе → Сохранить → Перезагрузить публичную страницу → Изменение отображается.
- Это работает только в том же браузере.
- Для публикации на сервере нужно экспортировать JSON и перенести данные в `lib/catalogModels.ts`, либо подключить CMS/backend.

---

## Экспорт / Импорт JSON

В нижней части редактора — раздел **«Дополнительно»** (сворачиваемый).

- **«Экспортировать JSON»** — показывает текущие данные в виде JSON.
- **«📋 Скопировать»** — копирует в буфер обмена для вставки в `lib/catalogModels.ts`.
- **«↩ Сбросить к данным проекта»** — очищает localStorage, возвращает исходные данные из кода.
- **Поле импорта** — вставьте JSON для применения изменений.

---

## Библиотека изображений (MinIO/S3)

В нижней части страницы — сворачиваемый блок **«Библиотека изображений»**.

Позволяет загружать изображения по папкам (Каталог, Брендирование, Hero, Портфолио), просматривать загруженные файлы, копировать URL, удалять файлы.

Требует запущенного MinIO-сервера (см. `docs/03-installation-and-local-run.md`).

---

## Безопасность

- `/admin/*` — `robots: noindex`, недоступно поисковикам
- `robots.txt` закрывает `/admin/` и `/api/admin/`
- Cookie передаётся только по HTTPS в production (`secure: true`)
- `sameSite: 'lax'` защищает от CSRF

## Вкладка «Портфолио»

### Структура кейса

| Блок | Описание |
|---|---|
| Главное изображение (cover) | Показывается в карточке на /portfolio/ и в верхней части страницы кейса |
| Галерея | Несколько фото внутри страницы кейса |
| Изделия в заказе | Список изделий с фото, описанием, характеристиками |

### Как добавить главное изображение кейса

1. Выберите кейс в левой колонке.
2. Найдите блок **«Главное изображение (cover)»**.
3. Введите URL или нажмите 📁 для загрузки.
4. Нажмите **«Сохранить кейс»**.
5. Изображение появится в карточке на `/portfolio/` и на странице кейса после обновления.

### Как добавить галерею

1. В форме кейса найдите блок **«Галерея изображений»**.
2. Нажмите 📁 (загрузить) или **«+ URL»** (вставить ссылку).
3. Каждое фото отображается с preview и кнопкой «Удалить».
4. Нажмите **«Сохранить кейс»**.

### Как добавить изделия в кейс

1. Прокрутите форму кейса до блока **«Изделия в заказе»**.
2. Нажмите **«+ Добавить изделие»**.
3. Заполните: название, описание, количество, материал, цвет, брендирование.
4. Добавьте фото через блок «Фото изделия» (поддерживает несколько фото).
5. Нажмите **«Сохранить изделие»** → затем **«Сохранить кейс»**.

### Что отображается на публичных страницах

| Страница | Что показывается |
|---|---|
| `/portfolio/` | coverImage или CSS-заглушка |
| `/portfolio/[slug]/` | coverImage, galleryImages, caseProducts с фото |

> **Для новых кейсов:** отдельная страница `/portfolio/new-slug/` не создаётся автоматически — нужен перенос данных в `lib/portfolio.ts`.

---

## Ограничения текущей реализации

| Ограничение | Решение |
|---|---|
| Изменения видны только в одном браузере | Экспортировать JSON → перенести в `lib/catalogModels.ts` / `lib/portfolio.ts` |
| Новые категории каталога не создают публичные страницы | Добавить route вручную или подключить CMS |
| Новые кейсы портфолио не создают публичные страницы | То же — перенести в `lib/portfolio.ts` |
| Нет истории изменений | Нужна база данных |
| Нет ролей/нескольких администраторов | Нужен backend с авторизацией |
