# 10. Форма заявки и обработка лидов

## Где находится форма

| Страница | Тип |
|---|---|
| Главная `/` | Секция внизу страницы, id=`request` |
| Контакты `/contacts/` | Встроенная в страницу, тот же компонент |
| Другие страницы | Кнопки «Рассчитать» ведут на `/#request` или `#request` |

Компонент: `components/RequestForm/RequestForm.tsx`

---

## Поля формы

Форма содержит три видимых поля:

| Поле | HTML name | Обязательное | Описание |
|---|---|---|---|
| Имя | `name` | **Да** | Имя или ФИО контактного лица |
| Телефон | `phone` | **Да** | Номер для связи |
| Email | `email` | Нет | Электронная почта |

Скрытые поля (не видны пользователю):
| Поле | HTML name | Описание |
|---|---|---|
| Конфигурация | `comment` | Автозаполняется из конфигуратора каталога (если выбран вариант) |
| Honeypot | `website` | Защита от ботов |

---

## Прикрепление файлов

Разрешённые форматы: PDF, PNG, JPG/JPEG, WEBP, SVG, TXT, DOC, DOCX, XLS, XLSX, ZIP

Максимальный размер: **10 МБ**

Файл прикрепляется к письму как вложение (при отправке через SMTP/Ethereal) или сохраняется в папке `.tmp/request-submissions/files/` (в dev-режиме).

---

## Honeypot-защита от спама

В форме есть скрытое поле `website` (невидимо для людей, но видно ботам). Если это поле заполнено при отправке — запрос молча отклоняется (возвращается `success: true`, но письмо не отправляется). Это стандартная защита от автоматических спам-ботов.

---

## Автозаполнение из конфигуратора

Если пользователь настроил параметры изделия в конфигураторе и перешёл к форме, поле «Комментарий» заполняется автоматически:

- **Многопозиционный расчёт** — читает `localStorage.teeon_quote_items` (массив позиций)
- **Выбранный вариант** — читает `localStorage.teeon_selected_variant`
- **Единичная конфигурация** — читает `localStorage.teeon_quote_selection`

После успешной отправки заявки все эти ключи удаляются из `localStorage`.

## API-эндпоинт формы

**URL:** `POST /api/request`  
**Файл:** `app/api/request/route.ts`

Что делает эндпоинт:
1. Читает FormData из запроса
2. Проверяет honeypot (поле `website`)
3. Валидирует обязательные поля (name, phone, product)
4. Проверяет файл (тип, размер)
5. Вызывает `sendLeadEmail()` из `lib/mail.ts`
6. Возвращает `{ success: true, message: '...' }` или ошибку

---

## Режимы отправки

Управляются переменной `LEAD_EMAIL_MODE` в `.env.local`.

### dev (по умолчанию)

Заявки сохраняются в JSON-файлы:
```
.tmp/request-submissions/
  1716300000000.json        ← данные заявки
  files/
    1716300000000-logo.pdf  ← прикреплённый файл
```

Email не отправляется. Идеально для разработки.

### ethereal

Отправка на тестовый почтовый сервис Ethereal Email. Не требует реальных SMTP-настроек.

В режиме разработки (`NODE_ENV !== 'production'`) ответ API содержит поле `previewUrl` — ссылку для просмотра письма в браузере. Письмо доступно несколько дней.

```json
{
  "success": true,
  "message": "Заявка принята!",
  "previewUrl": "https://ethereal.email/message/..."
}
```

### smtp

Отправка через реальный SMTP-сервер. Требует настроек:
```env
LEAD_EMAIL_MODE=smtp
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your@gmail.com
SMTP_PASS=пароль_приложения
SMTP_SECURE=false
LEAD_TO_EMAIL=manager@teeon.ru
LEAD_FROM_EMAIL=noreply@teeon.ru
```

---

## Как подключить production SMTP

### Gmail

1. Включить двухфакторную аутентификацию в Google-аккаунте
2. Создать «пароль приложения» в настройках Google → Безопасность → Пароли приложений
3. Использовать этот пароль как `SMTP_PASS` (не основной пароль аккаунта)

```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-address@gmail.com
SMTP_PASS=xxxx-xxxx-xxxx-xxxx   ← пароль приложения
SMTP_SECURE=false
```

### Yandex Mail

```env
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=your@yandex.ru
SMTP_PASS=пароль_приложения
SMTP_SECURE=true
```

### Любой другой провайдер

Уточните SMTP-настройки у провайдера и заполните соответствующие переменные.

## Как проверить форму

### Быстрая проверка через скрипт

```bash
# Запустить сервер разработки
npm run dev

# В другом терминале
npm run test:form
```

Скрипт отправляет 3 тестовых запроса и проверяет ответы. Результат выводится в консоль.

### E2E-тест в браузере

```bash
npm run test:e2e
```

Playwright автоматически заполняет и отправляет форму в реальном браузере.

### Ручная проверка

1. Открыть http://localhost:3000
2. Прокрутить к форме заявки
3. Заполнить обязательные поля: ФИО, Телефон, «Что нужно изготовить»
4. Прикрепить тестовый файл
5. Нажать «Получить расчёт»
6. Должно появиться сообщение «Заявка принята!»
7. В режиме `ethereal` — проверить ссылку previewUrl

## Структура письма

Письмо отправляется в двух форматах: HTML и plain text.

HTML-письмо содержит:
- Заголовок «Новая заявка с сайта TEEON»
- Таблицу с данными (ФИО, телефон, email, компания, способ связи, тема, тираж, срок, комментарий)
- Информацию о прикреплённом файле (если есть)

Файл прикрепляется как email-вложение (при smtp/ethereal).
