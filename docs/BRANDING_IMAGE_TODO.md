**Куда класть:** `public/branding-samples/<method>/<filename>`
**Как добавить через админку:** `/admin/catalog-editor/` → вкладка «Нанесение» → выбрать метод → редактировать вариант → загрузить файл в поле «Фото варианта». Файл уйдёт в `/uploads/branding/` и подставится автоматически — пути в `lib/brandingSamples.ts` менять не обязательно.

**Альтернатива (через файловую систему):** положить файл с указанным именем в указанную папку — путь уже прописан в `lib/brandingSamples.ts`, иконка появится без правок кода.

**Требования к файлам:**
- Формат: `.webp` (предпочтительно), допустим `.jpg`/`.png`. При смене расширения — обновить путь в `lib/brandingSamples.ts`.
- Ширина: 1200–1600 px
- Вес: до 600 KB
- Без водяных знаков, без чужих логотипов, без видимых лиц крупным планом без модельного релиза
- Не использовать фото конкурентов / чужие работы
- Лицензия: Unsplash / Pexels / Pixabay / Wikimedia (CC0, CC-BY, CC-BY-SA) / собственные фото TEEON
- После загрузки внести запись в `docs/BRANDING_IMAGE_SOURCES.md` и `public/branding-samples/sources.json`

## Что делать, если для варианта нет легального фото
Оставить `imageSrc` как есть (путь к файлу прописан, но файла нет) — на сайте отрисуется аккуратный placeholder «Фото не добавлено». Когда добавишь файл по нужному пути — он сразу подхватится без правок кода и пересборки.
